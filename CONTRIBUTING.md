# Contributing

1. Create a branch from `main`.
2. Keep SI units explicit in the engineering core.
3. Add a test for every changed calculation or rule.
4. Record method, equation, assumptions, limits and references.
5. Never present synthetic data as validation or a current quotation.
6. Run `make verify` before opening a pull request.

Engineering changes require review by a competent naval engineer. A passing
software test is not evidence of statutory compliance.
